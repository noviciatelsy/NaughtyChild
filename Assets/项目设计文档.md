## 1. 项目概览

- **引擎/管线**：Unity（URP，`com.unity.render-pipelines.universal 14.0.12`）
- **关键 Package**（`Packages/manifest.json`）：
  - `com.unity.inputsystem 1.14.2`（已使用 `PlayerInput.inputactions` 自动生成 `PlayerInput.cs`）
  - `com.unity.ai.navigation 1.1.7`（NPC、狗、司机均使用 NavMeshAgent）
  - `com.unity.cinemachine 2.10.7`（`CameraGroup.prefab`）
  - `com.unity.timeline 1.7.7` + `com.unity.modules.director`（开场 Timeline `StartCG.playable`）
  - `com.unity.textmeshpro 3.0.7`、`com.unity.visualeffectgraph 14.0.12`
  - `com.coplaydev.unity-mcp`（开发期 MCP 接入）
  - 第三方：DOTween / DOTweenPro（`Plugins/`，对外有专用 .csproj）
- **核心玩法**：俯视角 3D 解谜小品。玩家在「淘气小孩」回合中收集 cookie 通关，每完成一回合，会把违规的"待定规则"固化进 **规则板**；之后的回合不能再违反它们，否则失败重开。当生效规则数 ≥ `RulesToEnd` 时切到 `Ending` 场景。

## 2. 核心架构

整体采用 **单例 Manager + ScriptableObject 数据 + C# event** 的事件驱动模式。

```
                       ┌────────────────────┐
                       │   GameManager       │  (DontDestroyOnLoad)
                       │  状态机 / 回合 / 事件 │
                       └─────────┬──────────┘
                                 │ events
   ┌─────────────────────────────┼─────────────────────────────┐
   │                             │                             │
┌──────────────┐         ┌──────────────────┐          ┌──────────────────┐
│ RuleSystem   │         │ AchievementMgr   │          │  UI 层           │
│ 单例 + Rule SO│         │ 单例 + AchievSO  │          │ RulesBoard /     │
│ active/pending│         │ Bool/Int/Float   │          │ AchievementPopup │
└─────┬────────┘         └─────────┬────────┘          │ DialogueManager  │
      │                            │                    │ Settings(暂停)   │
      │TriggerRuleSystem           │RecordAction(name)  └──────────────────┘
      │                            │
┌──────────────────┐       ┌─────────────────────┐
│ Interact / RuleTrigger    │ playermovement / PlayerHand / Throwable │
└──────────────────┘       └─────────────────────┘
```

所有核心单例（`GameManager` / `RuleSystem` / `SceneTransition` / `SoundManager`）都在 `Awake` 里 `DontDestroyOnLoad`，并自带"重复销毁"保护。`AchievementManager` / `PlayerHand` / `DialogueManager` 是普通场景单例（不跨场景）。

### 2.1 状态枚举

[`GameState`](script/GameManager.cs#L8)：
`WaitingToStart`、`Playing`、`Paused`、`RoundComplete`、`GameOver`。

`SetState` 内部去重，仅在变化时触发 `OnGameStateChanged`。

### 2.2 事件总线（GameManager 暴露）

| 事件 | 触发时机 | 主要订阅者 |
|---|---|---|
| `OnGameStateChanged(GameState)` | `SetState` 变化 | – |
| `OnRoundStarted(int round)` | `StartRound()` | `Interact.OnRoundChanged` 重置物件 / `Dog.OnRoundStart` / `AddFence3.OnRoundChanged` |
| `OnRoundCompleted(int round)` | `CompleteRound()` | – |
| `OnRuleCommitted(Rule)` | 该轮 pending → active | `RulesBoard.AddRuleData` 在板上追加新规则 |
| `OnRuleViolated(Rule)` | `HandleRuleViolation` | `RulesBoard.OnRuleViolated` 高亮闪红 |
| `OnShowRulesRequested(bool)` | 玩家按 Tab 或 `Settings` 打开 | `RulesBoard.HandleToggle` / `Settings.OnShowRulesRequested` |
| `OnSwitchBoardRequested` | 玩家在 UIMode 按切换键 | `RulesBoard.SwitchContent` 规则↔成就页切换 |
| `OnCookieCollected(int)` | 通关一次 cookie++ | `RulesBoard.OnCookieCollected` 弹跳计数 |
| `OnGameRestarted` | `ResetGameState` | – |

`ClearAllEvents()` 在跨场景重开时被调用，避免旧场景对象残留导致空引用。

## 3. 回合 / 规则 / 成就系统

### 3.1 回合循环（`GameManager`）

[`GameManager.cs`](script/GameManager.cs) 关键流程：

1. `Awake` 单例化 + `DontDestroyOnLoad`。
2. `Start`：`RuleSystem.Instance.Initialize()`，订阅 `SceneManager.sceneLoaded`。若 `_director`（Timeline）正在播放（PlayOnAwake），开协程 `WaitForDirectorThenStart`，等播完 → `RestartRound()`。
3. `OnSceneLoaded(GameScene)`：重新播放 Timeline 后再 `RestartRound()`。
4. `StartRound()`：`CurrentRound++`、`RuleSystem.ResetPending()`、`SetState(Playing)`、`SetplayerPos()`（找 `Player` tag 重置 Rigidbody 速度，`(0, 0.5, -2)`；同时调用所有 `car.ResetCarState()`）。
5. **通关**：玩家拾取 / 触发 `Cookie.cs` → `CompleteRound()`：
   - 切 `RoundComplete`，`CookieCount++`、广播 `OnCookieCollected`。
   - `RuleSystem.CommitRound()` 把当前 pending 规则提升为 active，广播 `OnRuleCommitted`。
   - 若 `RuleSystem.GetActiveRules().Count >= RulesToEnd` ⇒ `TransitionToEnding()` 用 `SceneTransition` 切到 `Ending`。
   - 否则继续 `StartRound()`。
6. **违规**：`HandleRuleViolation(Rule)` 暂停（`Time.timeScale = 0`，`SetState(Paused)`）+ 广播 `OnRuleViolated` + 强制打开规则板，等 `violationDisplayDuration`（默认 2s 真实时间）后 `RestartRound()`。
7. **跨场景重开**：`ResetGameState()` 清规则、回合归零、清事件，再由目标场景的 Manager 自行 `StartRound()`。

> 字段 `RulesToEnd` 默认 `1`（在脚本里），但典型用法在 Inspector 里调高（例如 5/10），决定累计多少条规则后进入 Ending。

### 3.2 规则系统（`RuleSystem` + `Rule` SO）

- [`Rule.cs (RuleType.cs)`](script/Rules/RuleType.cs)：纯数据 ScriptableObject，字段 `priority` + `description`；通过 SO 资产 `name` 作唯一 key。`OnRuleViolated(violator)` 直接转发到 `GameManager.HandleRuleViolation`。
- [`RuleSystem.cs`](script/Rules/RuleSystem.cs)：
  - `allRules` 全集（Inspector 拖入），构建 `ruleMap` (string→Rule)。
  - `initialRules` 在 `Initialize` 时直接 `ActivateRule`。
  - `pendingRule`：本回合内能"候选"上的最高 priority；`SetPending(name)` 比较 priority 决定是否替换。
  - `CommitRound()` 把 `pendingRule` 加入 `activeRules` 并广播 `OnRuleActivated`。
  - 提供 `IsRuleActive`、`GetActiveRules`、`ClearAllRules`。
- 触发入口：
  - `Interact.TriggerRuleSystem(name)`（受保护方法，被各 Interact 子类调用）。
  - `RuleTrigger`（`Collider trigger`）：玩家进入即检查；如已 active → 触发违反；否则 `SetPending`。同时支持挂 `AchievementSO` 一并 `RecordAction`。
  - 子类 `AddFence3`：当指定规则在新一轮变 active 时显示其引用的 `fence` 物体（"蓝字补丁"——只允许第一次触发）。

#### 已配置的规则资产 `Assets/script/Rules/Data/`

```
askdogforhelp1 / 2 / 3   # 找狗帮忙的多阶段链
AskNPCForHelp / 2        # NPC 给斧子 / 苹果
DontDestroydoor          # 禁止用斧砍门
DontDestroyfence         # 禁止砍栏
DontJustOpendoor         # 禁止用手开门
DontstandOntree          # 不要站在倒下的树上
DontStepOnGrass          # 不要踩草
Donttouchboard / box / rock
DontUsewell              # 用井传送一次后变 active，再用即违反
drivecar                 # 开车（一次后再开为违反）
breakbox2                # 砍储物箱
Farjump / NPChead        # 远跳 / 踩 NPC 头
secretpath1 / secretpath2
touchcar                 # 被车撞过一次后再撞为违反
```

> 这些是 SO 资产名，与脚本里 `IsRuleActive("…")` 字符串严格对应；改名前必须同步替换所有出现处。

### 3.3 成就系统（`AchievementManager` + 三型 SO）

- 基类 [`AchievementSO`](script/Achievements/AchievementSO.cs)：`icon / achievementName / description`，`IsUnlocked` + `OnUnlocked` 事件。`TryUnlock` 检查 `CheckUnlock`（子类实现）。
- 三种实现：
  - `BoolAchievementSO`：`List<bool> conditions`，全部 true 时解锁，可分多步。
  - `IntAchievementSO`：累计 `currentValue >= targetValue`。
  - `FloatAchievementSO`：浮点累计版本。
- `AchievementManager.RecordAction(actionName, amount=1)`：根据 SO 类型 dispatch。`Start` 时 reset 所有进度并订阅每个成就的 `OnUnlocked` → 转发 `OnAchievementUnlocked`。
- 已配置成就 `Assets/script/Achievements/Data/`：`axe / Gooddog / Guide / Jessie / KingofChild / high / YouHaveSteppedOnTheGrass`。
- 被动触发位置：
  - `playermovement.CheckFallOutOfWorld`（掉出地图）。
  - `Dog.StartGoToDoor / TriggerRules`（依赖 `askdogforhelp` 链状态）。
  - `NPCchild.OnInteracted`（用斧/苹果让 NPC 帮忙、`NPChead`）。
  - `RuleTrigger`（草地等）。
  - `ice.OnInteracted`（拾取司机掉的冰块）。

## 4. 玩家 / 输入 / 物理

### 4.1 输入资产

`Assets/PlayerInput.inputactions` 通过 `Generate C# Class` 输出 [`PlayerInput.cs`](PlayerInput.cs)，含两个 Map：

- **GameMode**：`Move (WASD 2D Vector)`、`Interact (E)`、`isRushing`、`Jump (Space)`、`ShowRules (Tab)`。
- **UIMode**：在打开规则板时启用，至少包含 `OnSwitchBoard` / `OnCloseBoard`。

`playermovement` 同时实现 `IGameModeActions` 与 `IUIModeActions`，`SetBoardInputState(true/false)` 在两个 Map 间手动切换 callback 与 enable。

### 4.2 玩家移动 [`playermovement.cs`](script/playermovement.cs)

- 组件：`Rigidbody`（`freezeRotation = true`、`Continuous` 碰撞、Interpolate）。
- 状态枚举 `PlayerState { Normal, Driving }`，序列化引用 `playerVisual / carVisual / playerCollider / carCollider / drivingTrigger`，进入车后通过 `SetDrivingVisual(true)` 切换可见/碰撞/质量（`playerMass=10` ↔ `carMass=100`）。
- `Move_Normal`：相机 yaw 投影到水平面，构造 forward/right；冲刺乘 `sprintMultiplier`；空中检测 `IsHittingWall` 防止贴墙；只覆盖 X/Z 速度，保留 Y。
- `ApplyFallAcceleration`：下落时额外 `1.2x` 重力，提升手感。
- `HandleJump`：地面单段跳，`v.y = sqrt(jumpHeight * -2g)`。
- `HandleMouseDrag`：左键按住拖动 → 旋转 `cameraRoot` 与 `playertextureroot` 的 Y 轴；同时通过 `holdThreshold (0.5s)` 区分"拖视角"与"点击交互"。
- **交互射线（核心）**：`OnInteract` 在 `canceled` 且按键时长 < holdThreshold 时触发：
  1. 屏幕鼠标 → `mainCamera.ScreenPointToRay`。
  2. `Physics.SphereCastAll(ray, 0.3f, 300f)` 收集所有 `Interact`。
  3. 仅保留距玩家 ≤ 3m 且 `Interactable == true` 的目标。
  4. 评分：`alignment*2 - distance*0.2`，取最高分。
  5. 行为分支：
     - `PlayerHand.HasItem == true` + 命中 → `UseItem(target)`（如对箱子用斧）。
     - 有物品但未命中 → 朝鼠标射线命中点（再次 raycast）方向 `ThrowItem`。
     - 空手 + 命中 → `Interact.InteractObject`。
     - 空手 + 未命中 → 无操作。
- `EnterCar / ExitCar / UpdateCarPhysics`：
  - 进入时禁用 `targetCar.gameObject` 并把车 Visual/Collider 切到玩家身上。
  - 驾驶状态下不靠 Rigidbody 推动，而是 `transform.position += inputDir * carMoveForce * dt`，朝向用 Slerp 平滑。
  - `OnTriggerEnter` 在驾驶下撞到 door / fence / Box / tree → 调用各自 `BreakByCar()`。
- `ApplyExternalForce(velocity)`：被 car 撞击时 `car.OnTriggerEnter` 调用，瞬时冲量 + `externalForceTimer`（0.3s）期间禁止主动移动。
- `CheckFallOutOfWorld`：`y < fallDeathY (-20)` → 一次性触发"高空掉出"成就 + `RestartRound`。

### 4.3 手与投掷

- [`PlayerHand.cs`](script/PlayerHand.cs)：单例。`PickItem(Throwable)` → 设置父级到 `handPoint`，本地坐标归零；`UseItem(target)` 把当前物品转给目标 `Interact`；`ThrowItem(dir)` 调 `Throwable.OnThrow`；`ResetHand` 用于回合重置。
- [`Throwable.cs`](script/Throwable.cs)：`RequireComponent<Rigidbody, LineRenderer>`。
  - 拾起：`isKinematic = true`、禁用所有自身 Collider、`SetIgnorePlayerCollision(true)`。
  - **轨迹预测**：每帧 `DrawTrajectory()`：取鼠标射线命中点，按距离归一化在 `[0.15, 2.0]s` 内插值得到飞行总时长，反推初速度 `cachedVelocity`，按 `resolution=150` 个 timeStep `0.02s` 在 `start + v0*t + 0.5*g*t²` 上采样，遇到非自身/非玩家碰撞提前截断 LineRenderer。LineRenderer 渐变白→透明。
  - 释放：恢复 `useGravity`，`rb.velocity = cachedVelocity`，`RestoreCollision(0.3s)` 后恢复与玩家碰撞，避免卡身体。
  - `OnUse(target)` 默认调 `target.GetComponent<Interact>().InteractObject(this.gameObject)`，把自己作为道具传入。

## 5. 交互物（`Assets/script/Interact/`）

[`Interact.cs`](script/Interact/Interact.cs) 基类：
- 字段：`ruleName`、`interactable`、`interactHint`，`Interactable` 还要求 `GameState == Playing`。
- `Start` 缓存 `col / rend / rb` 与初始 transform，订阅 `GameManager.OnRoundStarted -> OnRoundChanged -> Reset()`，所以**所有交互物在每轮自动复位**。
- `InteractObject(item)` 通用流程：若 `item` 上有 `Throwable` 且玩家空手 → `PlayerHand.PickItem(item)`；再调子类 `OnInteracted`；返回 success 才广播 `OnInteractedEvent`。
- `TriggerRuleSystem(name)`：active 则 `rule.OnRuleViolated(this)`，否则 `SetPending`。

### 5.1 各子类要点

| 脚本 | 行为摘要 |
|---|---|
| `apple / axe / board` | 仅作为可拾取/投掷的标签类（空实现），靠 `Throwable` 提供物理与轨迹 |
| `Cookie` | 通关物：`InteractObject` 后播 `小孩糖笑` 音效，触发 `CookieFlyEffect.Fly(... → CompleteRound())`；无特效时直接 `CompleteRound()`。`Reset` 时重置飞行状态 |
| `Box` | 仅当 `item` 为 `axe` 时被破坏：禁用 collider/renderer，按 `woodCount` 在圆周上 Instantiate `woodPrefab`，播 `木头破坏2`；`Reset` 销毁所有掉落物。可被 `BreakByCar` 触发 |
| `Box2`（储物箱）| 重写 `InteractObject`：`axe` ⇒ `BreakBox`（先 `TriggerRuleSystem("breakbox2")`，已 active 则违反返回）；空手 ⇒ `Function1` 在 `spawnPoint` 周围 Instantiate `spawnPrefab`（用于"敲击掉物"） |
| `car` | 自动循环 A→B 速度移动；玩家用斧砍后 `canDrive=true`，停车并 `SpawnDriver(driverPrefab→driverSpawnPoint)`，把 `Driver.targetPoint` 设成 `DtargetPoint`；玩家 E 互动后调 `playermovement.EnterCar`，并触发 `drivecar` 规则。`OnTriggerEnter` 把玩家击退 `force` 力 + `PlaySquashEffect` 用 DOTween 做"压扁→拉伸→OutBounce"三段缩放。`Reset/ResetCarState` 销毁司机、回 A 点 |
| `cookie.cs` | 见上 |
| `Dog` | 行为见 §6.1 |
| `door` | 用斧砍：触发 `DontDestroydoor`，活跃则违反；否则 `Breakfence()` 隐藏 + 掉木板。空手：触发 `DontJustOpendoor`，活跃则违反；否则 `OpenDoor` 协程，0.5s 旋转到 `(0,-90,0)` 打开。`Open()` 给狗/NPC 用，**不**触发规则。`BreakByCar` 同 `Breakfence` |
| `Driver` | 自动 NavMeshAgent 到 `targetPoint`，并 `SpawnIceAfterDelay(1s)` 掉一块冰；到达后 `Destroy(gameObject)`（外加距离判定的 `Arrive` 兜底） |
| `fence` | 仅斧子可破，触发 `DontDestroyfence`；其他物品返回 false，不消耗 |
| `HeadBounce` | 挂在 NPC 小孩头部子物体的 trigger：玩家落入且 `y > 0.65`（视为跳起）时 `AddForce(Vector3.up * bounceForce, VelocityChange)` 把玩家反弹，并对 child 做 DOTween "压扁→OutBounce" 动画，调用 `npcchild.NPChead()` 触发 `NPChead` 规则 + 同名成就 |
| `ice` | `oneShot`：被互动一次记录 `AchievementSO`（典型为 `KingofChild` 之类），然后 `Destroy` |
| `NPCchild` | 详见 §6.2 |
| `tree` | 砍树多阶段：第 1 击 `FallTree`（0.3s 协程把 transform 插值到 `fallenTransform`）+ 掉 2 个 `apple`；2~4 击仅累计；第 5 击 `DestroyTree` + 掉 `boardPrefab` x3。倒下后 `OnTriggerEnter Player` 触发 `DontstandOntree`。`BreakByCar` 直接倒树 |
| `well` | 用斧子破井盖。否则若 `DontUsewell` 已 active → 违反；否则 `SetPending("DontUsewell")` 并 `TransportPlayer` 把玩家 Rigidbody 移到 `transpoint`，并通知 `linkedWell.OnPlayerTeleported()` 在出口井 `d1` 上播 0.35s 翻盖动画 |

### 5.2 `RotateY.cs` / `SpindleRotate`

文件名是 `RotateY.cs` 但里面类是 `SpindleRotate`：每帧 Y 轴自转 + Z 轴 `Sin` 摆动，给奖励物（如 cookie）一种"纺锤旋转"展示效果。

## 6. AI 角色

### 6.1 狗 [`Dog.cs`](script/Interact/Dog.cs)

NavMeshAgent，3 大状态变量：`isLeashed / isSleep / actionState{Idle, GoingToDoor}`。

- `SyncStageFromRules`（`OnRoundStart` 触发）：
  - 任意 `askdogforhelp1/2/3` active ⇒ `isLeashed = true`。
  - `askdogforhelp2/3` active ⇒ `isSleep = true`。
- 交互优先级：用斧 → 解 leash；用苹果 → 解 sleep（同时 `appleFx.Play`）；否则若 sleeping/leashed 阻断；否则 `StartGoToDoor()` + `TriggerRules`（按 1→2→3 顺序逐级推进）。
- `Update`：广告牌朝相机；`GoingToDoor` 时到达 `doorReachDistance` 调 `door.Open()`；`isSleep` 时停 agent 且 `SleepZEmitter` 飘 Z；否则按 leashed/free 在 `wanderCenter` 半径内 `HandleWander`；leash 期间 `LineRenderer` 黑线连接到 `leashedPoint`。
- 三段都解后会推 `Gooddog` 成就。

### 6.2 NPC 小孩 [`NPCchild.cs`](script/Interact/NPCchild.cs)

NavMeshAgent，`NPCState{Idle, GoingToDoor, Done}`：

- `Idle`：在 `startPoint` 半径内 `wanderRadius` NavMesh 随机点游走，到点等 `idleWaitTime` 再换点。
- 交互：用斧 → `AskNPCForHelp`、用苹果 → `AskNPCForHelp2`，记 `achievement`，状态切到 `GoingToDoor`，到 `doorApproachPoint` 调 `door.Open()` 然后回 `Idle`。
- 精灵广告牌：`spriteTransform` 反向 LookAt 主相机水平向。

## 8. UI 设计

### 8.1 规则板 [`RulesBoard.cs`](script/UIHandler/RulesBoard.cs)

挂在 `prefab/UI/Board 1.prefab`：
- 子物体：`Panel`（半透明遮罩，`Image`），`content`（`RectTransform` + `CanvasGroup` + `ScrollRect`），`cookieCountText` (TMPro) + `cookieIcon`，两个锚点 `ort`/`drt`（隐藏/展开 X 坐标）。
- `rulePrefab = RulePrefab.prefab`、`achievementPrefab = AchievementPrefab.prefab`。
- 订阅 `GameManager.OnRuleCommitted / OnShowRulesRequested / OnSwitchBoardRequested / OnRuleViolated / OnCookieCollected` 与 `AchievementManager.OnAchievementUnlocked`。
- `HandleToggle(true)`：DOTween 同时执行 ① Panel `Image` `DOFade(0.5)`，② 自身 Image `DOFade(dtp)`，③ `selfRect.DOAnchorPosX` 滑入到 `drt.x`（`OutCubic`），④ `DOPunchScale 0.15`。`SetUpdate(true)` 不受 `Time.timeScale=0` 影响。
- `HandleToggle(false)` 反向滑出，并切回规则页、清高亮。
- `SwitchContent`：CanvasGroup 0.15s `DOFade(0)+DOScale(0.9)` → 切换 ruleInstances/achievementInstances active + 切换 sprite 背景 → 再 `DOFade(1)+DOScale(1, OutBack)`。
- `OnRuleViolated`：自动切回规则页，`DOFade(0.3, 0.3s, Yoyo, 无限)` 让违反的规则文字闪红，并通过 `ScrollToItem` 让目标条目滚到视口中点（DOTween `verticalNormalizedPosition`）。
- `OnCookieCollected`：cookie 文字 + 图标各做 `DOPunchScale`。

### 8.2 暂停菜单 [`Settings.cs`](script/UIHandler/Settings.cs)

挂在 `prefab/UI/Settings.prefab`：
- 按钮：`pauseButton / resumeButton / mainMenuButton / quitButton / showRulesButton`，`volumeSlider` 直接绑定 `AudioListener.volume`。
- ESC 键：① 若正在 board 模式 → 关 board；② 否则若菜单已开 → 关；③ 否则 `Playing` → `OpenPauseMenu`。
- 打开/关闭：`pausePanel.SetActive` + CanvasGroup `DOFade` + `DOScale 0.9↔1 (OutBack/InCubic)`，并 `GameManager.Pause/Resume` 控制 `Time.timeScale`。
- 「显示规则」按钮：`_waitingForBoardClose=true`，淡出菜单，`RequestShowRules(true)` 并 `playermovement.SetBoardInputState(true)` 切到 UIMode；规则板关闭时 `OnShowRulesRequested(false)` 回 Resume。

### 8.3 成就弹窗 [`AchievementPopup.cs`](script/UIHandler/AchievementPopup.cs)

- `OnAchievementUnlocked` → 入队 `_queue`，单协程串行播放：滑入 `OutBack(slideTime)` → 停留 `displayTime` → 滑出 `InCubic(slideTime)`。所有 Tween `SetUpdate(true)`；附带 `unlockSound` AudioSource。

### 8.4 对白 [`DialogueManager.cs`](script/UIHandler/DialogueManager.cs)

### 8.5 开始 / 结束场景

## 11. 关键开发约定 / 易踩坑

1. **规则名是字符串契约**：脚本里的 `IsRuleActive("...")` 与 `Assets/script/Rules/Data/*.asset` 的 SO 资产名必须严格一致；改名前需要全局搜索。
2. **Tag**：玩家必须 `tag = "Player"`，多处 `FindWithTag("Player")` 与 `CompareTag("Player")` 依赖此设置。
3. **回合复位**：所有 `Interact` 子类要把"破坏后产生的对象 / 状态变量"在 `Reset()` 里清掉，否则下一回合会出现幽灵物体；现有 `Box / Box2 / door / tree / fence / well` 都遵守这个模式。
4. **跨场景单例**：`GameManager / RuleSystem / SceneTransition / SoundManager` 都做了"已存在则销毁"的去重；新增类似 manager 时务必沿用。
5. **`Time.timeScale`**：暂停期间所有提示/转场/规则板 Tween 必须 `SetUpdate(true)`，否则会卡住。`SceneTransition.cs / RulesBoard.cs / Settings.cs / AchievementPopup.cs` 都已遵守。
6. **`SetBoardInputState`** 必须由 `Settings` 与 `playermovement.OpenBoard/CloseBoard` 任一入口调用，确保 GameMode/UIMode 不会同时启用。
7. **PlayerInput 重新生成**：修改 `PlayerInput.inputactions` 后会覆盖 `PlayerInput.cs`，`playermovement` 会在编译期检测接口变动。
8. **物理质量切换**：进入车辆会把 Rigidbody mass 从 10 → 100，对场景中不希望被推动的物体要预留质量/约束。
9. **NavMesh**：每个 GroundPrefab 都有独立烘焙的 NavMesh 数据；新增地形需要烘焙后才能让狗 / NPC / Driver 找路。
10. **`DialogueBehaviour` 当前硬编码台词**：使用 Signal 的 `SetText` 是更灵活的入口；后续维护时建议把 `dialogueText` 字段真正生效（删去硬编码字符串）。
11. **`Throwable.Start`** 里把 `lr = GetComponent<LineRenderer>()` 覆盖了 `Awake` 里的 `GetComponentInChildren` 结果——若 LineRenderer 挂在子物体上，会变 null。新增可投物体时建议把 LineRenderer 挂在根物体或调整该行。
12. **`car.PlaySquashEffect`** 用 DOTween Sequence，`OnKill` 又再次 `DOKill(false)` 可能导致递归——目前未观察到崩溃，但改动时需谨慎。
